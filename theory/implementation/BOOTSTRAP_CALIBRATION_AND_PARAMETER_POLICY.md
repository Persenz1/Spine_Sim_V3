# 开发期标定与参数替代政策

**版本：** 0.1.0-dev
**日期：** 2026-07-17
**适用阶段：** 求解器开发、回归测试、灵敏度分析和直线拖曳实验设计
**禁止用途：** 把临时参数解释为目标红砖/混凝土的真实微观参数，或据此认证 C 级临界承载

## 1. 这份政策替你做出的决定

目前不等待白光轮廓仪，也不等待完整材料参数。开发按下面的主线继续：

> **先用解析/程序化合成表面、刚性单边接触、Coulomb 摩擦、针梁与真实安装弹簧、no_damage，完成 A/B 求解器；再用唯一可做的直线拖曳实验标定少量“联合有效输出参数”，而不是反演一整套不可辨识的微观参数。**

同时冻结以下边界：

1. A/B 的无损伤开发和趋势探索可以立即进行。
2. 真实壁面 PSD、局部强度、断裂能、接触刚度和摩擦系数不填“最佳猜测单值”，而是使用明确标记的宽先验/扫描。
3. 数值容差、网格、步长和随机样本数通过收敛决定，不通过实验拟合。
4. C 的非零 +X、45°、rocking 和正式 \(F_{crit}\) 在 B 2.x 前保持不可用。
5. 直线拖曳曲线只用于拟合阵列/单刺的联合响应；不把 \(F_x/P_z\) 直接当摩擦系数。
6. 所有临时结果必须带 `DEV_PRIOR`、`synthetic_surface`、`no_damage`、`not_certifiable` 等 provenance 标签。

配套机器可读配置见 [DEV_BOOTSTRAP_PROFILE.yaml](DEV_BOOTSTRAP_PROFILE.yaml)。

## 2. 为什么不能“把所有未知参数都拟合出来”

实验只观测一条标量曲线

\[
y(x)=F_x(x).
\]

但模型里有表面高度场、局部法向、摩擦、接触柔度、针梁柔度、安装弹簧、材料强度、损伤历史和逐针状态等许多未知量。不同参数组合可能给出几乎同一条 \(F_x(x)\)：

\[
(\mu,\text{坡度},k_c,k_b,k_s,T_0,G_c,\ldots)
\longrightarrow F_x(x).
\]

这个多对一映射意味着“曲线拟合得很好”不等于“每个参数都找对了”。例如上升段的斜率近似来自串联柔顺：

\[
\frac{1}{k_{eff}}
\approx
\frac{1}{k_{machine}}
+\frac{1}{k_{fixture}}
+\frac{1}{k_{beam}}
+\frac{1}{k_{spring}}
+\frac{1}{k_{contact}}.
\]

只观测 \(k_{eff}\) 时，不能唯一拆出每一项。类似地，较大拉力可能来自较大摩擦、较有利的局部坡度、较深互锁或较强材料，单通道数据无法把它们分开。

因此本项目采用：

- 已知几何和材料弹性：固定或给窄范围；
- 数值参数：收敛确定；
- 不可辨识的微观物理：给宽先验并做灵敏度；
- 实验真正可见的组合量：直接标定；
- 当前合同不能表达的自由度：明确拒绝，不用经验数“补洞”。

## 3. 参数分级与最终处置

### 3.1 绿灯：现在就固定或解析计算

| 项目 | 决定 | 来源/理由 |
|---|---|---|
| 坐标与符号 | 使用当前正式工程事实 | 不需要实验 |
| 针尖半径 | 50、100 µm 两档 | 已固定几何 |
| 针直径 | 0.6、0.8 mm | 已固定几何 |
| 暴露长度 | 基准 4 mm；梯度设计按补偿公式 | 已固定几何 |
| 安装角 | 50/60/70/80° 与既定梯度 | 已固定设计空间 |
| 阵列 | \(n_x,n_y=2\ldots6\)，间距 4/5/6 mm | 已固定设计空间 |
| 单刺主动推力 | 0.5 N | 已固定工况 |
| B/C 每单元主动推力 | 0.5/1.0/1.5/2.0 N 扫描 | 覆盖已固定范围 |
| 拖速与行程 | 1 mm/s、100 mm | 已固定实验主线 |
| 弹簧行程 | 0–4 mm | 已固定机械约束 |
| 梁截面量 | 由 d、E 解析计算 | 不从总曲线反拟合 |
| 旋转、wrench、偏心矩 | 解析计算 | 已通过独立代数检查 |

制造公差应保留为输入分布，不要因为 CAD 是单值就假设实物无公差。

### 3.2 黄灯：用开发先验和宽扫描，暂不称为标定

| 项目 | 开发决定 | 解释边界 |
|---|---|---|
| 钢弹性模量 E | 基准 210 GPa；扫描 200/205/210 GPa | 手册级弹性先验 |
| 泊松比 ν | 基准 0.30；扫描 0.28/0.29/0.30 | 手册级先验 |
| 摩擦系数 μ | 基准 0.40；扫描 0.15/0.25/0.40/0.60/0.80 | 只做敏感性，不代表红砖实测 |
| 弹簧刚度 | 0.1/0.2/0.5/1.0/2.0 N/mm | 覆盖固定工程范围；有供应商值时以其为中心 |
| 表面 Hurst 指数 H | 基准 0.7；扫描 0.5/0.7/0.9 | 无材料标签的合成粗糙面 |
| RMS 高度尺度 | 用 \(S_q/R_t=0.25,1,4\) | 采用无量纲覆盖，不伪称真实 Sq |
| 相关尺度 | 用 \(\ell_c/R_t=5,20,80\) | 同上 |
| 各向异性比 | 1、2 | 只测试方向敏感性 |
| 随机种子 | 公共随机数，逐级加样 | 保证方案比较方差较小 |

这些值来自工程数量级、文献覆盖与计算需要，不是“隐含的真实材料参数”。文献也明确提示随机粗糙表面参数需要对目标表面重新标定：[随机粗糙接触证据卡](../../archive/web_pro_derivation_2026-07-17/docs/extract/机理提取/11_randomly_rough_contact_mechanics/evidence_card.md)、[红色陶瓷粗糙度证据卡](../../archive/web_pro_derivation_2026-07-17/docs/extract/机理提取/12_red_ceramic_blocks_roughness/evidence_card.md)。

### 3.3 红灯：当前不得编造物理单值

| 项目 | 当前状态 | 为什么 |
|---|---|---|
| 真实红砖/混凝土 3D PSD、局部法向与曲率 | unavailable | 单轴拖曳不能重建 3D 形貌 |
| 真实局部接触刚度 | unavailable | 与梁、弹簧、夹具、机器柔顺混淆 |
| 唯一摩擦系数 | unavailable | 与局部坡度和几何互锁混淆 |
| \(c,\phi,f_t,G_c,A,\ell,r_D,\rho\) | unavailable | 宏观试件参数不能直接迁移到针尖微区 |
| 针体屈服/极限强度与根部 SCF | unavailable for certification | 无材料证书与根部几何应力映射 |
| 每根针真实载荷 | unavailable | 无逐针传感器 |
| DamageStore 空间核 | unavailable | 标量曲线不足以唯一识别 |
| C 的 local-y、转动、6D tangent | contract unavailable | 是理论/软件接口问题，不是标定问题 |
| rocking 稳定性与正式 \(F_{crit}\) | unavailable | 需要 B 2.x 和六维验证 |

宏观砖材 Mode-I 强度/断裂能可以用作数量级警戒或独立材料回归，但不能直接代入针尖压碎/剪裂：[砖/凝灰岩断裂证据卡](../../archive/web_pro_derivation_2026-07-17/docs/extract/机理提取/14_fired_clay_tuff_fracture_properties/evidence_card.md)、[多尺度砖材证据卡](../../archive/web_pro_derivation_2026-07-17/docs/extract/机理提取/15_fired_clay_multiscale/evidence_card.md)。

## 4. 不等白光轮廓仪的表面方案

### 4.1 三层表面库

第一层用于公式和代码回归：

- 平面；
- 单斜坡；
- 正弦波；
- 单球冠/单凹坑；
- 双峰和多峰；
- 有已知最近点切换的组合曲面。

第二层用于事件和载荷共享测试：

- 程序化 multi-bump；
- 有意构造的台阶、尖峰、鞍点和沟槽；
- 可制造粘着→滑动→释放→再挂接顺序的表面。

第三层用于设计敏感性：

- 无材料标签的自仿射 Gaussian 随机场；
- 参数为 \((S_q/R_t,\ell_c/R_t,H,a_{ani},seed)\)；
- 每个设计使用相同 seed 集合，即 common random numbers。

### 4.2 不允许的说法

没有形貌测量时，不允许：

- 把一个合成高度场命名为“真实红砖表面”；
- 只凭 Ra 生成唯一 3D 表面并声称重建成功；
- 用二维照片或单条轮廓恢复唯一 3D 法向/曲率；
- 以某篇论文的砖面参数替代当前样品而不做不确定性声明。

### 4.3 分辨率决定

局部查询初始网格建议

\[
\Delta=R_t/5,
\]

并在 \(R_t/8\)、\(R_t/10\) 复算。只有当力曲线摘要、事件位置和方案排序收敛后才接受。

100–150 mm 全域不能统一使用数微米密集网格；应使用：

- 解析/程序化 evaluator；
- 按需生成的 tile；
- 局部细化；
- 多分辨率缓存。

这保留 150×150 mm 查询域，又避免数亿至十亿网格点。

## 5. 接触、结构和损伤的开发主线

### 5.1 接触

主线选用刚性 Signorini 接触和 Coulomb 摩擦：

\[
g_n\ge0,\quad \lambda_n\ge0,\quad g_n\lambda_n=0,
\]

\[
\|\boldsymbol\lambda_t\|\le\mu\lambda_n.
\]

理由是梁与安装弹簧已经提供物理柔顺，不必为了“能算”给壁面虚构接触刚度。

若实现罚接触：

- 罚刚度只能标为 numerical regularization；
- 必须做罚参数收敛；
- 不得把它报告为壁面真实 \(k_c\)。

### 5.2 结构

主排序基线使用：

- needle bending = on；
- 独立弹簧按真实设计分支；
- 刚性安装使用真正的约束图/广义方程。

`bending=off + rigid mount` 只用于理想图回归或上下界；若集合值求解器未实现，应返回 unavailable，不能用“大刚度”假扮刚体后进入正式排名。

### 5.3 损伤

阶段 1：

\[
\texttt{material\_model=no\_damage}.
\]

它可以验证几何、摩擦、梁/弹簧、活动集、载荷共享和事务。它会高估失效后的持续承载，因此只能用于亚失效行为和开发回归。

阶段 2 仅用于测试失效代码路径：

- 先修复 [复核报告](../review/DERIVATION_VERIFICATION_2026-07-17.md) 中损伤末端与功共轭问题；
- 使用 `resultant_capacity` 的无量纲 `DEV_PRIOR` fixture；
- 例如容量/主动推力比扫描 2/4/8、log 标准差 0.5、残余比 0/0.2；
- 不进入设计认证排序；
- 不解释成砖材 \(c,\phi,G_c\)。

阶段 3 才考虑目标材料损伤：

- 需要额外的局部证据或多通道实验；
- 必须给出尺度桥、混合模态和能量共轭；
- 当前单轴拖曳数据不够。

## 6. 数值参数由收敛决定

下面是首轮开发起点，不是永久规范：

| 参数 | 起点 |
|---|---:|
| 初始拖曳步长 | \(R_t/2\) |
| 最大步长 | \(R_t\) |
| 最小步长 | \(R_t/1000\) |
| 事件位置容差 | \(R_t/100\) |
| 力残量绝对容差 | \(10^{-6}\) N |
| 力残量相对容差 | \(10^{-5}\) |
| Newton 最大迭代 | 50 |
| 事件括区间最大迭代 | 80 |
| 同位置级联上限 | 50 |

接受数值配置需要同时通过：

1. 解析平面/坡面/球冠回归；
2. 步长减半，关键摘要变化低于预设阈值；
3. 表面 \(R_t/5\to R_t/8\to R_t/10\) 收敛；
4. 事件不因步长改变而换序；
5. 功、残量和 complementarity 误差随精化下降；
6. 排名在随机样本增加后稳定。

随机样本建议：

- smoke：4 seeds；
- 日常开发：16 seeds；
- 初筛：64 seeds；
- 最终探索：64→128→256→512 顺序加倍；
- 当主要排名和置信区间连续两级稳定后停止。

## 7. 直线拖曳实验究竟能标定什么

### 7.1 可以直接保存的原始量

- 时间 \(t\)；
- 编码器实际位置 \(x\)；
- 原始拉力 \(F_x\)；
- 主动推力命令 \(P_z\)；
- 设计、针/阵列、壁面批次和轨迹 ID；
- 拖曳方向；
- 新轨迹或复用轨迹；
- 传感器校准版本。

原始力永久保留。滤波、基线校正和事件检测只能生成有版本号的派生通道。

### 7.2 可辨识的联合量

1. **首次挂接位置的生存分布**

   对 100 mm 内未挂接的样本做右删失。优先报告非参数生存/风险曲线。只有主动接受“立即命中 + 指数搜索”模型时，才拟合

   \[
   P(X>x)=(1-p_0)e^{-\lambda x}.
   \]

   文献中的 \(p_0,\lambda\) 只能做回归 fixture，目标壁面仍需重估：[随机柔顺阵列证据卡](../../archive/web_pro_derivation_2026-07-17/docs/extract/机理提取/09_stochastic_compliant_spine_arrays/evidence_card.md)。

2. **有效加载链刚度**

   在平滑、已挂接、无明显事件的上升段拟合

   \[
   k_{eff}=\frac{dF_x}{dx}.
   \]

   它是机器—夹具—针梁—弹簧—接触的组合，不拆分成单项。

3. **有效容量分布**

   记录条件峰值、分位数、峰值前位移、掉落幅度和峰后保持率。它表示“给定装置、表面、方向与预载下”的 resultant capacity，不等于唯一材料强度。

4. **事件与再挂接统计**

   记录上升、跌落、恢复、释放、再挂接的空间间隔，以及

   \[
   W_+=\int_0^{100\,mm}\max(F_x,0)\,dx.
   \]

5. **重复轨迹的经验退化**

   同一轨迹第 \(m\) 次与第一次比较：

   \[
   r_m=\frac{Q_m}{Q_1},
   \]

   其中 Q 可取峰值、正功或事件数。它只能称为经验保持率/退化率，不能唯一解释成断裂能。

6. **趋势与排序**

   可比较针尖、阵列、角度、刚柔、主动推力和方向的输出分布。项目现阶段应优先追求“排序在宽先验下是否稳定”，而不是假精确的绝对承载值。

### 7.3 不能由单轴曲线唯一识别

- 真实 3D PSD、坡度、曲率和局部法向；
- μ 与几何互锁的各自贡献；
- 接触/梁/弹簧/夹具的各自刚度；
- 局部强度、断裂能和损伤核；
- 摩擦耗散与材料耗散的各自份额；
- 每根针的状态与载荷；
- CoP、作用线、自由力偶和完整 6D wrench；
- C 的 rocking 与正式 \(F_{crit}\)。

尤其不能把

\[
\mu=F_x/P_i
\]

作为测量结果，因为 \(P_i\) 是主动推力命令，不是实测总法向接触力，而且拉力中含几何互锁和结构作用。

## 8. 在“仍然只有一种实验能力”前提下的试验设计

下列变化仍属于同一种“直线模组拖曳 + 拉力记录”实验，不要求白光轮廓仪。

### 8.1 台架基线

如果设备允许：

1. 空载跑完整行程，测零点、漂移和位置相关背景；
2. 用已知弹簧或刚性挡块估计整机加载链柔顺；
3. 传感器每批实验前后做零点检查。

这些是同一台架的校准工况，不是新的材料实验。

### 8.2 单刺优先

若夹具可更换，先测单刺，再测 B 单元：

- 单刺固定 \(P_z=0.5\) N；
- 新轨迹用于挂接和容量统计；
- 同轨迹重复用于退化统计；
- 单刺与阵列不要混在同一参数层拟合。

如果只能装完整阵列，就只标定“阵列级有效挂接风险率和容量”，不得把它误称为单刺参数。

### 8.3 B 单元因素

首轮建议：

- \(P_z=0.5,1.0,1.5,2.0\) N；
- 刚性与独立弹簧方案；
- 代表性的 2×5 与 5×2；
- 0°、45°、90° 和反向拖曳；
- 不同壁面区域作为独立 block。

旋转试样或夹具仍是直线拖曳，只用于得到经验方向响应，不能反演唯一 PSD。

### 8.4 重复数与停止规则

每个因素组合：

1. 先做 12 条独立新轨迹；
2. 之后每批增加 8 条；
3. 对峰值、首次挂接距离和方案排名做 bootstrap；
4. 当连续两批的区间宽度和排序基本稳定时停止；
5. 同一轨迹的重复通过不能当独立样本。

少量三五次曲线可以调试设备，不能支撑尾部承载或可靠性结论。

### 8.5 训练/验证划分

按完整轨迹或壁面位置划分训练与留出集。禁止把同一条高采样率曲线的点随机拆成训练集和验证集，否则会产生严重的数据泄漏。

## 9. 建议拟合的最小有效参数包

不拟合十几个微观参数，首版只拟合：

\[
\theta_{eff}
=\{S_X(x),\ k_{eff},\ \mathcal D(F_{peak}),\
\mathcal D(\Delta x_{event}),\ r_{repeat}\}.
\]

其中：

- \(S_X(x)\)：首次挂接距离生存函数；
- \(k_{eff}\)：挂接段组合刚度；
- \(\mathcal D(F_{peak})\)：条件峰值分布；
- \(\mathcal D(\Delta x_{event})\)：事件/再挂接间距分布；
- \(r_{repeat}\)：重复轨迹经验保持率。

若使用参数化 hazard，可再用 \((p_0,\lambda)\)；若证据不支持指数模型，保留非参数曲线，不强行压成两个数。

仿真校准采用摘要统计或联合后验：

\[
p(\theta_{eff}\mid y)
\propto p(y\mid\theta_{eff})p(\theta_{eff}),
\]

而不是声称得到唯一 \(\mu,S_q,G_c\)。

## 10. C 层的明确决定

### 10.1 当前实现内容

论文与后续正式物理主线选择 `C-R`：预紧阶段 s 是唯一对称径向机构自由度，各单元 Z 运动由共同刚体 pose 和 s 导出；\(s_{stop}\) 锁定后才形成固定刚体关系，不把四个 \(u_{z_i}\) 当作彼此独立自由度。C-R 调 B 的 `PRESCRIBED_XZ_RESIDUAL`，\(P_i\) 只有在端点/作用线绑定后才作为 C 级外载。现有逐单元 `UX_PZ_BALANCED` 只作为 `C-I` 条件备选；只有实际机构确有四个独立 Z 行程，且执行器端点、作用线和功均已冻结时才启用。该 proposed 选择需在下一次正式模型修订中版本化。

- 完整请求/响应 schema；
- 合同覆盖检查；
- 非法自由度的 `C_CONTRACT_EXTENSION_REQUIRED`；
- 零状态、零历史、零曲线推进回归；
- 纯 Z、零增量和解析 mock；
- `prescribed-s scan`，输出每一步原始特征。

解析 mock 必须隔离为 `not_certifiable/not_comparable`，不能进入设计排序。

### 10.2 当前不做自动锁定

在实验数据到位前，C1 只做规定 s 扫描，不自动宣称“预紧充分”。以后可令

\[
s_{max}
=\min(
\text{机械/碰撞安全行程},
\text{单元达到平台距离的高分位}
).
\]

平台、最弱分支、保持和下降否决门必须用独立轨迹留出集确定。不能回退使用 100 mm 搜索距离，也不能照搬别的论文的 4.5 mm。

### 10.3 升级门

- B 2.0 固定姿态三平移后，才可做 rocking-off 的 +X/45°；
- 完整 SE(3)/6D B 后，才可做 rocking；
- 法向执行器端点与真实边界冻结后，才可认证执行器功和六维平衡；
- 以上完成并数值/实验验证后，才可输出正式 \(F_{crit}\)。

## 11. 分阶段实施门

| G0 理论修订 | 修 A 姿态、损伤末端，使 B 弹簧摘要服从 A 零位分支并冻结事件策略 | P0 项有版本化合同 |

## 12. 可直接执行的最短路径

1. 先修复复核报告中的 P0-A1，并按 A 权威分支修正 P0-B1 的 B 摘要冲突；同时把损伤主线关闭。
2. 实现 A 的解析表面、有限球尖、Signorini/Coulomb 和 EB。
3. 实现 B 的公共 \(u_z\) 平衡、活动集、曲线路径事件定位和事务。
4. 用配套 YAML 跑解析回归和 16-seed 合成表面开发集。
5. 同步建立原始拖曳数据格式；不要等仿真“完成”才开始规范数据。
6. 先用实验拟合最小有效参数包，验证设计趋势。
7. 把 B 2.x 作为独立的软件/理论任务，不把它列为“等实验标定”。

这样做可以让编程立即向前推进，同时把尚无证据的部分诚实地保留为不确定性，而不是用一个漂亮但不可验证的参数表制造虚假确定性。
