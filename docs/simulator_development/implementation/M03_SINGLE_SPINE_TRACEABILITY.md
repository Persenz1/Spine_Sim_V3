# M03 SINGLE_SPINE 实施与验收追踪

**任务：** `M03_SINGLE_SPINE_IMPLEMENTATION`

**冻结需求：** [`M03_SINGLE_SPINE_REQUIREMENTS 1.0.0`](../requirements/M03_SINGLE_SPINE_REQUIREMENTS.md)

**实施状态：** implementation delivered / acceptance blocked / evidence incomplete

**范围标签：** `DEV_PRIOR / synthetic_or_analytic_surface / no_damage / not_certifiable`

M03 已交付 A-M0 typed contracts、本征核、standalone 外层驱动、M00 result extension、解析几何验证、campaign 编排、摘要和只读验证图入口。这里的“交付”只表示代码面已形成，不表示 §20 已完成，也不表示模块 `ACCEPTED`。artifact-capable 本征 standalone 路径已经将 normal-bearing 的 non-active/full-candidate raw 与真实 M00 receipt 一并发布，并覆盖 fault rollback、idempotency 与 semantic replay；独立梯形 actuator input work 和 14 类本征 runtime guards 也已经实现。最终 medium 36 + gentle/sharp 2 已全部执行并形成明确 capability termination，但 0 个完成 100 mm、0 个 final M00 receipt、0 个趋势值。退化 `normal=None` 候选的 frozen-schema 表示、真实功闭合、最终变形几何重查询和完整事件路径仍未闭合；M04、M05、M06 未开始。

## 状态口径

- `CLOSED`：本行限定的合同/代码/测试证据已闭合；不单独提升 M03 总体验收状态。
- `PARTIAL`：已有实现和局部测试，但冻结矩阵中的系统级或负例证据不完整。
- `BLOCKED`：缺少 §20 必需的实跑或独立闭合证据，当前不能判为完成。

## 最小兼容修复与解释边界

- M00 shaped numeric vector 的 validation/storage 兼容修复只允许支持 M03 声明的 `[2]`/`[3]`/`[6]` 数值 shape；不得改变 scalar、identity、transaction 或 replay 语义。
- embedded validation demo 与注入式 response-only standalone fixture 都可生成真实 M00 receipt；前者不是完整 standalone，后者按合同没有 full-candidate artifacts。本征 artifact-backed fixture 已保存 normal-bearing 的完整候选 raw；`normal=None` 退化候选仍明确不可表示。
- 软件和解析 fixture 只能证明声明范围内的代码与数值行为。所有 M03 结果保持 `experimentally_validated=NOT_ASSESSED`、`not_certifiable=true`；damage、fracture energy、needle strength certification 均 unavailable/not applicable，root resultants 不是安全系数。
- M04、M05、M06 未开始，也不得用 M03 campaign/recipe 边界提前实现它们。
