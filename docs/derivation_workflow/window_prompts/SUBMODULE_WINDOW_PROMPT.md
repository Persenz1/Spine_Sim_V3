# 子模块单任务窗口启动提示词

> 用法：把下方“提示词正文”完整复制到一个新的 Codex 窗口，只修改第一行的 `TASK_ID`。  
> 允许值：`A1`、`A2`、`A3`、`B1`、`B2`、`B3`、`C1`、`C2`、`C3`。  
> 本文件是 Codex 窗口启动提示词，不上传给网页端 Pro。

## 提示词正文

```text
TASK_ID: A2

本窗口只完成 TASK_ID 指定的一个子模块任务，不得自动开始下一子模块、大模块集成或全局集成。整个任务在本窗口分为“网页执行前准备”和“网页产物回收”两个阶段。

我只负责两类人工操作：
1. 按你最终给出的精确清单，把文件和正式执行提示词上传到网页端；
2. 把网页端生成的文件下载后原样回传到本窗口。

除此以外，提示词编写、任务轮次、文件路径、上传清单、输入清单、哈希、YAML、diff、模板适配、校验、归档、快照、Git 提交和推送都由你完成。不要让我参与机械排障；只有无法从权威项目文件唯一确定的工程事实或物理机理选择才可以问我。

开始时先读取仓库顶层 AGENTS.md，并严格执行其中两个工作流入口。

如果本消息没有附带网页输出文件，执行“阶段一：网页执行前准备”：

1. 完整阅读 PROMPT_AUTHORING_GUIDE.md 及其要求的全部文件，尤其是最新工程事实、MECHANISM_MODULE_PLAN、LITERATURE_MODULE_ROUTING、DERIVATION_WORKFLOW、四个输出模板，以及本任务所需的当前 MODULE_CONTEXT 或上游合同。
2. 根据 TASK_ID 自动确定模块、前置结果、最终最小文献包和下一可用运行编号 <TASK>-rNN；不得为轮次、路径、模板或文件名询问我。
3. 检查前置模块是否已经接受、所需文件是否存在、工程事实生成器是否通过。机械缺失自行修复；只有缺少不可替代的语义前置结果时才停止并说明。
4. 编写或更新 TASK_ID 的正式网页端执行提示词。提示词必须针对钩爪式爬壁机器人爪刺啮合研究，显式写出研究背景、当前模块位置、全部准确上传路径、阅读优先级、任务边界、必答问题、证据边界、完成判据、四个输出文件和自检要求，不得退化成通用占位模板。
5. 按 RUN_ARTIFACT_HANDLING_GUIDE.md 提前建立本轮不会覆盖历史的运行目录，保存 PROMPT.md 和 INPUT_MANIFEST.yaml，记录 run_id、实际目录、输入 Git 提交与逐文件 SHA-256，并运行工程事实基线校验。
6. 检查正式提示词中的上传清单、INPUT_MANIFEST.yaml 和实际文件完全一致，然后提交并推送本阶段准备文件到已核验的 origin/main。
7. 最终只向我交付：TASK_ID、run_id、可点击的正式提示词路径、按网页上传顺序编号的完整文件清单，以及网页端应下载的四个文件名。清单中每个文件必须给出准确路径；不要让我自行从工作流推断。
8. 交付后停止，等待我回传网页输出，不得提前编写下一任务。

网页端子模块固定回收四个文件：
- 当前大模块最新完整的 {A|B|C}_MODULE_CONTEXT.md；
- ENGINEERING_FIXED_CONTEXT_CANDIDATE.md；
- RUN_UPDATE_SUMMARY.yaml；
- CITATION_BRIEF.md。

如果本消息或后续消息已经附带网页回答和输出文件，执行“阶段二：网页产物回收”：

1. 不重新生成提示词，不要求我重复任务背景；读取 RUN_ARTIFACT_HANDLING_GUIDE.md 及其全部必读输入，并使用阶段一分配的 run_id。若窗口中没有可用 run_id，则扫描已有运行清单自动确定，不询问我机械编号。
2. 原样保存 RAW_RESPONSE.md 和网页候选文件；所有修复只作用于拆分后的工作副本，并记录 MECHANICAL_FIXES。
3. 自行完成 YAML 解析、候选与正式工程事实完整 diff、模板/路径修复、引用检查、模块语义边界检查、工程事实生成器校验和必要的可复现验证。
4. 若没有实质语义冲突，将通过审查的完整 MODULE_CONTEXT 更新为 current 并保存历史快照；CITATION_BRIEF 只在本轮本地归档，不进入下一任务上传链。
5. 若存在工程事实变化或两种物理机理必须人工取舍，只向我报告自然语言形式的“当前事实、候选变化、证据、影响和需要决定的选项”，不要展示 YAML 排障过程。
6. 完成归档后提交并推送到已核验的 origin/main；最终报告接受状态、验证结果、关键语义变化、提交号和文件路径。
7. 到此结束本窗口任务，不得自动开始下一子模块或集成任务。
```
