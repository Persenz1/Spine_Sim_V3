# `{A|B|C}_MODULE_CONTEXT` 最低内容合同

> 本文件只规定滚动模块上下文必须包含的内容，不强制统一的论文排版、章节长度或表格数量。
>
> A1、B1、C1 输出对应大模块的首版完整上下文；A2/A3、B2/B3、C2/C3 必须输出融合此前已接受内容后的最新完整上下文，不能只输出本轮增量。

## 文件头必须说明

- 大模块与当前完成阶段；
- 上下文版本；
- 使用的工程事实版本和提示词版本；
- 当前状态：`candidate` 或 `accepted`。

## 1. 范围与边界

必须说明：

- 当前完整上下文已经覆盖哪些子模块和物理问题；
- 本模块明确不处理什么；
- 哪些问题留给本大模块后续子模块；
- 哪些问题属于其他大模块，禁止在本层越权定义；
- 必须遵守的工程事实 ID。

## 2. 坐标、符号与单位

必须定义后续方程和接口实际使用的：

- 坐标系、正方向、角度约定；
- 标量、向量、矩阵和集合符号；
- 每个量的单位；
- 与工程固定上下文同名符号的对应关系。

不得让同一符号在不同阶段静默改变含义。

## 3. 输入、输出与状态

必须列出：

- 必需输入及其类型、单位和前置条件；
- 输出及下游用途；
- 当前状态变量和历史变量；
- 初值、更新时机和持久范围；
- 数据模型或接口中不可缺少的字段。

若当前阶段没有历史变量，明确写明原因。

## 4. 机理、方程与约束

必须给出足以实现和审查的：

- 模型对象和基本定义；
- 控制方程、几何关系或平衡关系；
- 每个方程的变量、假设、适用条件和单位；
- 单边约束、可行性条件、边界、限位或碰撞条件；
- 主模型与可选分支的启用条件；
- 文献不能直接迁移时采用的重推、参数化或标定处理。

不要求固定排版，但不得只给概念描述而缺少可执行关系。

## 5. 状态转换、事件与计算流程

必须说明：

- 状态或活动集合如何定义；
- 事件函数和触发条件；
- 事件发生后的状态更新；
- 同时事件的优先级或冲突处理；
- 回退、减步长、无解和终止条件；
- 能够转化为程序的单步计算顺序。

纯几何阶段也应定义首次接触、碰撞等几何事件。

## 6. 参数、证据与标定状态

每个关键参数或模型选择必须区分：

- 正式工程事实；
- 本地上传文献证据；
- 外部网络或公开数据库证据；
- GPT 自身通用知识；
- 本轮推导量；
- 必须实验标定或仍未解决的量。

必须说明证据适用对象、尺度、迁移边界和冲突。不得把定性趋势升级为未经标定的定量定律，也不得伪造来源。

## 7. 验证、完成判据与未决问题

必须包含：

- 解析极限、数值收敛或表示一致性检查；
- 对当前子模块完成判据的逐项核对；
- 已知证据缺口和模型风险；
- 未决问题、临时占位方式和关闭条件。

不能因问题尚未解决而将其静默删除。

## 8. 对下一阶段的交接

必须明确：

- 下一阶段可直接调用的变量、状态和接口；
- 必须原样继承的定义；
- 可以修改但必须显式说明的内容；
- 禁止下游重新定义的低层机理；
- 下一阶段仍需补齐的问题。

## 输出前最低自检

- 是否为当前大模块的最新完整上下文，而非本轮增量；
- 工程事实与机理实现是否分离；
- 符号、单位、状态和事件是否闭合；
- 方程是否给出假设、约束和适用范围；
- 参数是否标明来源与标定状态；
- 未决问题和下游交接是否保留。
