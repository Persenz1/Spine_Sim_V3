# 29 篇文献的模块路由台账

> 用途：为 `MECHANISM_MODULE_PLAN.md` 的 A1–C3 九个小模块分配完整文献压缩包。
> 约束：不再从文献提取二次片段；实际交付单位是每篇文献对应的完整 `.zip`。同一篇可跨模块使用，但每个模块应尽量只保留少量、直接且互补的压缩包，为网页端推理留出上下文。

## 1. 路由规则

每篇完整阅读后同时记录两种归属：

- **可用模块**：论文确有证据能够支撑的所有小模块，不因上下文预算删除；
- **建议上传模块**：考虑整包上传成本后，论文值得实际占用上下文名额的模块。

最终精简遵循：

1. 直接给出方程、状态模型或实现流程的论文优先于仅有趋势的论文；
2. 能在同一模块回答多个关键问题的论文优先；
3. 同一篇跨模块重复分配必须有独立且实质性的作用；
4. 仅有边缘旁证时，记录为“可用”，不自动进入建议上传包；
5. 全部 29 篇都必须完整阅读，但不要求全部进入九个模块的最终上传清单；
6. 最终清单形成后，再做一次模块内去重、证据缺口检查和跨模块复核。

- **B2–C3**：没有多刺载荷共享、整爪同步预紧或整体力/力矩平衡证据，不分配。

#### 使用边界

- 摩擦系数、搜索分布参数、等效刚度及候选点折减率均需针对目标表面标定；
- 论文的 MI 符号描述存在内部矛盾，不应直接编码；
- 有限半径/峰宽阈值来自理想算例，不能作为红砖、混凝土或任意三维表面的普适常数；
- 泊松首次啮合模型不能直接扩展成阵列独立成功率。

- 阅读状态：已完整阅读

- **建议上传模块**：B2

- 在固定总法向载荷下展示刚性硬基底只有约 2–3 根实际接触，具体接触成员随拖曳更换；

- 软硅胶穿入不能外推为硬脆红砖失效，刚性阵列结论也不能直接外推到独立柔顺阵列。

### 09 — Stochastic models of compliant spine arrays for rough surface grasping

- 压缩包：`09_stochastic_compliant_spine_arrays/09_stochastic_compliant_spine_arrays.zip`
- 阅读状态：已完整阅读
- 证据性质：理论 + 随机模型 + 实验；单向柔顺阵列、递归再挂接和对置预载阵列
- **可用模块**：A1（中）、A2（中）、A3（强）、B1（强）、B2（强）、B3（强）、C1（强）、C2（中）、C3（中）
- **建议上传模块**：B3、C1

#### 对 B3 的作用（核心）

- 把行程离散为递归层，明确“保持、滑脱、未挂接、滑脱后重新进入搜索”的状态转移；
- 将挂接位置、剩余弹簧行程、线性加载和强度生存概率组合成完整力—位移统计骨架；
- 实验说明弱表面上再挂接贡献不可忽略。

#### 对 C1 的作用（核心）

- 两侧切向内力相减、离面分量相加；
- 中间预载弹簧与两侧刺柔顺串联，并由内/外限位约束共同位移；
- 说明增大预载提高预先挂接数和法向能力，却消耗净切向余量；软弹簧 + 初始力 + 硬止挡可形成近恒力预载。

#### 不扩展为 B2/C2/C3 核心的原因

- B2 的期望积分把各刺视为独立，明确忽略失效后的动态增载、相关失效与级联；
- 对置模型忽略横向载荷、不均匀挂接力矩、左右表面差异和单侧失效后的动态转移；
- 因而只作为 C1 预紧骨架，不承担偏心力矩或整体渐进失效。
