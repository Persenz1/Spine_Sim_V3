## 1. 研究背景与系统位置

本项目研究钩爪式爬壁机器人在红砖、混凝土和不同目数砂纸等硬质粗糙墙面上的爪刺啮合。机器人整爪由四个相同阵列爪单元按十字形式安装在无质量、无限刚性的理想参考体上；四个单元由共同驱动器同步沿各自局部正 x 向中心搜索，形成内部预紧后锁定共同径向位置，再在离十字参考中心正 Z 方向 50 mm 的加载点承受墙面内偏心位移加载。整爪必须通过四个历史相关阵列单元的 contact-only wrench、作用线、状态、事件和共享损伤共同满足完整力/力矩平衡，不能用四个单元峰值相加、固定分配矩阵或隐藏框架反力替代。

完整物理依赖链固定为：

表面查询 → 单刺算子 A → 阵列爪单元算子 B → 十字对爪算子 C。

当前任务只集成大模块 C。C1 已建立四单元坐标与参考点运输、唯一共同搜索坐标、内部预紧、最早/同时事件、共享 DamageStore、联合停止与 `C1PreloadState`；C2 已建立加载点与四单元刚体运动学、六维 contact-only wrench 装配、固定姿态/小角度 rocking、无隐藏支承、稳定性以及 B 运动覆盖审计；C3 已建立针级事件向单元退化传播、历史相关状态压缩、事件后四单元完整重平衡、渐进剥离、稳定反力曲线、非光滑/多峰能力、峰后路径和 `F_crit` 确认条件。

三个子模块已经接受，但顺序完成不等于大模块已经集成。本轮必须把它们重构为一个状态唯一、方程闭合、符号和单位统一、功账本一致、事件与事务可执行、没有重复柔顺或重复失效、能够明确说明当前认证边界的正式整爪模型。不得只拼接、压缩或摘要 `C_MODULE_CONTEXT` 的 48 个历史章节。

与本轮直接相关的固定工程背景包括：

- 墙面名义平面为全局 X-Y，正 Z 从墙面指向背板和自由空间；四单元局部 z 与全局 Z 共用。
- 单元局部正 x 从头部指向根部，也是向十字中心搜索/拖拽方向；局部 `e_y=E_Z×e_x`，四个局部系均为右手系。
- 四单元中线分别沿全局 `±X,±Y`，工程安装参考点满足 `O_i=C-40 e_xi mm`；中央 80 mm×80 mm 空区只提供力臂，不与墙面接触。
- B 公共参考点 `O_A` 是阵列未加载针尖球心规则格点的几何中心，不得默认等于工程参考点 `O_i`；偏置、变换和哈希必须显式保存。
- 四单元搜索阶段只有一个共同径向坐标 `s`；停止后 `s_stop` 锁定，不得给四个单元分配独立径向位移，也不得在偏心加载前归零或重新搜索。
- 每单元法向主动推力 `P_i` 各自处于 0.5–2 N；恒定的是主动广义推力，不是实际 contact-only 法向合力，更不是逐针载荷。
- 正式加载点为 `r_P=50 E_Z mm`；正式比较全局 `+X` 与从 `+X` 向 `+Y` 偏 45° 两个方向。
- 正式求解采用加载点准静态位移推进，输出反力—位移曲线、四单元载荷分配、压紧/剥离、针级/单元级渐进失效、峰值与失去稳定平衡的临界状态。
- 整爪参考体可选择 `rocking=off/on`；on 只允许绕全局 X、Y 的小角度摇摆，首版不扫描绕 Z 转动、不允许大角度倾覆，也不允许外部框架、导轨或连接件替爪刺接触承担载荷。
- 一次连续预紧—承载过程保留轻量不可逆损伤、事件和低层历史；新独立试验或新表面样本才从无损状态开始。
- 首版排除框架/导轨/传动链柔性、真实结构有限元、惯性动力学、大角度脱落后运动、显式裂纹、碎屑、重网格化和针尖磨损。
- C1 停止阈值与 `s_max`、B 完整 twist/姿态扩展、执行器端点/作用线、`eta_i` 自由度、rocking 有效域、材料/表面/损伤参数、事件与稳定性容差、显著退化阈值、峰值/分支策略、C3 加载速度和验证误差仍是未决项。

## 2. 本轮唯一目标与必须保留的认证结论

输出一个正式候选版 `C_INTEGRATED_MODEL.md`，统一定义从四单元同步搜索、预紧停止、径向锁定，到偏心位移加载、整体平衡、事件后重分配、稳定反力曲线和临界能力的完整 C 算子：

\[
\mathcal G_C:
\left(
\text{four accepted B states},
\text{shared DamageStore},
\text{common search/load path},
\{P_i\},
\text{rigid pose},
\text{external loading},
\text{history/configuration}
\right)
\mapsto
\left(
\text{accepted C state},
\text{reaction wrench},
\text{unit load distribution},
\text{events/degradation},
\text{capacity/status}
\right).
\]

同一个正式 C 模型必须包含两个连续但边界不同的阶段：

1. `synchronous_preload_stage`：在唯一共同 `s` 上调用四个无副作用 B trial，完成事件归约、损伤协调、预紧质量/最弱分支/安全门控和原子提交，生成唯一预紧接受态；
2. `eccentric_progressive_load_stage`：从该预紧接受态无损继续，锁定 `s_stop`，按 `delta_P` 推进完整刚体/单元运动、六维平衡、事件后四单元重平衡、稳定性、峰值和终止。

必须保留以下当前认证结论，不能在集成中“修好”或静默删除：

- `B_TO_C 1.0.0 accepted` 只认证单元局部 x 与全局 Z 平移。正式非零 `+X` 路径使 ±Y 单元出现局部 y 平移；45° 路径使四单元均出现局部 y 平移；真实 rocking 还需要动态姿态、针轴、表面和碰撞更新。
- 因此，在 B 2.x 等完整 twist/姿态扩展实现并被正式接受前，任何非零正式偏心加载必须返回统一的 `C_CONTRACT_EXTENSION_REQUIRED`（可映射历史 C2/C3 同义码），保留最后有效接受状态，不推进 `delta_P`、四个 B/A 状态、DamageStore、事件、功、曲线或峰值账本。
- 该安全拒绝不是零承载、物理无解、物理失稳、不可恢复脱附或 `F_crit=0`。C 的理论集成完成与当前在线正式加载未认证必须同时、清楚地报告。
- 本轮不得把 C_MODULE_CONTEXT 中提出的 `B_TO_C 2.x` 最小扩展要求改写为已经接受的新上游合同；只能形成版本化扩展要求、兼容条件和关闭验证。
