## 1. 研究背景与当前模块位置

本项目研究**钩爪式爬壁机器人在硬质粗糙墙面上的爪刺啮合机理**。有限半径球形针尖在红砖、混凝土和不同目数砂纸表面上连续搜索凸起；单刺会经历几何接近、接触、摩擦承载、针梁与独立轴向弹簧变形、滑移、局部材料失效、脱离和再挂接；规则阵列单元在共同刚性背板和每单元恒定法向主动推力下形成部分接触、非均匀载荷共享、失效后重分配和连续再挂接。最终机构由四个相同阵列单元沿全局 `±X、±Y` 组成十字主动对爪，先通过唯一共同径向坐标同步收紧形成内部预紧，再在离墙加载点承受偏心墙面内载荷。

完整物理依赖链为：

$$
\text{表面几何}
\rightarrow
\text{单刺历史状态}
\rightarrow
\text{阵列活动接触集与载荷共享}
\rightarrow
\text{十字同步预紧}
\rightarrow
\text{偏心六维平衡}
\rightarrow
\text{整体渐进失效与最大承载}.
$$

大模块 A 已冻结单刺接触、摩擦、针梁、轴向弹簧、材料失效、损伤、释放和再挂接的层级边界；大模块 B 已冻结阵列几何、共同运动、恒每单元法向主动推力平衡、逐针载荷共享、事件后重分配、共享 DamageStore 和历史相关单元响应。C 只能通过已接受的 `B_TO_C_CONTRACT 1.0.0` 调用 B，不得直接调用 A，也不得在整爪层重写逐针摩擦、材料、弹簧、损伤、活动集或载荷转移规则。

已接受 C1 建立了四单元同步搜索、内部预紧、停止后的 `C1PreloadState`、跨单元事件、共享损伤和全局原子事务。已接受 C2 在此基础上建立了：

- `C1PreloadState` 到 `C2AcceptedState` 的无损继承；
- 十字参考点 `C`、加载点 `P` 和四个 B 规范参考点 `O_A` 的刚体运动学与 wrench–twist 功对偶；
- `+X` 和 `45°` 两个正式方向、离墙 `50 mm` 偏心加载点及六维外部 wrench；
- 四单元 contact-only wrench 的六维装配、`80 mm` 中央空区提供的力臂和无隐藏支承原则；
- `rocking=off/on` 的统一受约束平衡、`theta_X/theta_Y` 法向位置变化、一侧增量稳定性、事件/损伤/事务流程；
- 当前 `B_TO_C 1.0.0` 只认证单元局部 x 与全局 Z 平移，不能认证正式 `+X/45°` 路径所需的局部 y 和姿态更新；因此在 B 2.x 扩展实现和接受前，正式在线加载必须返回 `C2_CONTRACT_EXTENSION_REQUIRED`，不能推进历史，也不能伪装为零承载或物理失效。

C3 是大模块 C 的第三个且最后一个子模块，位于 C1 同步预紧和 C2 偏心平衡之后、独立的大模块 C 集成之前。本轮必须从**当前完整 `C2AcceptedState`** 继续，建立针级事件向单元能力退化的传播、四单元递归再平衡、渐进剥离、稳定反力曲线、最大承载和峰后终止。完成 C3 仍不等于完成大模块 C 集成；不得在本轮自动编写 `C_INTEGRATED_MODEL.md`、启动 C 集成或开始全局集成。

与本轮直接相关的固定工程背景为：

- 墙面名义平面为全局 `X-Y`，`+Z` 从墙面指向背板；十字参考中心为 `C=(0,0,z_C)`；
- 四个单元局部 `+x_i` 指向十字中心，正式方向为 `+X、-X、+Y、-Y`，且 `e_{y_i}=E_Z×e_{x_i}`；
- 工程安装参考点满足 `O_i=C-40 e_{x_i} mm`，对置根部背板中点距离和中央空区边长均为 `80 mm`；B 规范参考点 `O_A` 与 `O_i` 的偏置必须使用 C1/C2 已接受的版本化变换，不能默认同点；
- C1 停止后共同内部径向坐标锁定在 `s_stop`。C2/C3 的整体刚体运动不得重置或重新推进四个不同的径向搜索坐标；
- 每个单元分别施加 `P_i∈[0.5,2] N` 的恒定法向主动推力。恒定的是执行器广义推力，不是实际墙面接触合力，不是整爪总推力，也不得除以针数；
- 外载作用点固定为 `P=C+[0,0,50]^T mm`；正式比较全局 `+X` 与从 `+X` 向 `+Y` 偏 `45°` 的两个墙面内方向；
- 正式求解采用准静态加载点位移推进，输出整体反力—加载点位移曲线、峰值可承受外力、四单元载荷分配、一侧压紧/对侧剥离、单元级和针级渐进失效以及失去稳定平衡的临界状态；
- 四单元相对无质量、无限刚性的理想参考体保持安装关系；`rocking=on` 只允许绕全局 X/Y 的小角度。首版不扫描绕 Z 扭转，不做大角度翻转、惯性动力学、框架/导轨柔性或真实传动链刚度；
- 外部框架、导轨和连接件不得替爪刺接触承担外载或力矩。固定姿态、禁用 yaw、数值正则化或约束乘子不能提供未授权承载；
- B 返回的 `A_on_B` contact-only wrench 已包含低层接触、针梁、轴向弹簧和硬限位反力。整爪只能运输并装配一次；`P_i`、加载执行器和其他控制/约束反力必须分栏；
- 一次连续承载过程保留轻量损伤记忆；新的独立试验或随机表面样本才从无损状态开始；
- 当前不固定二元成功阈值、综合评分、材料/表面参数、数值容差、显著退化阈值或 rocking 有效角度上限。缺失量必须保持参数化、待标定或未认证状态。

本轮不是一般性的“某个爪失效后把力平均分给其余爪”，也不是“求四个单元峰值之和”。输出必须形成历史相关、事件驱动、可实现且可审计的 C3 算子；每一次失效、损伤、姿态变化或活动集切换都必须由合法 B 完整回调和整体平衡重新求解，而不是由高层经验分配替代。

## 2. 本轮唯一目标

建立 C3 渐进承载试探与终止算子：

$$
\boxed{
\mathcal P_{C3}:
\left(
\mathcal S_{C2,n},
\Delta\delta_P,
\hat{\mathbf d},
\{P_i\}_{i=1}^{4},
\mathcal D_n,
\Theta_{\mathrm{event}},
\Theta_{\mathrm{stability}},
\Theta_{\mathrm{peak}}
\right)
\mapsto
\left(
\mathcal S_{C2,n+1}^{\mathrm{trial/accepted}},
F_{\mathrm{reaction}},
\mathcal E_{C3},
\mathcal M_{C3},
\Sigma_{C3}
\right)
}
$$

其中：

- `S_C2,n` 是完整 `C2AcceptedState`，包含当前加载路径、整体位姿、四个 B/A accepted states、共享 DamageStore、wrench、graph、事件、功、哈希和提交收据；
- `Delta delta_P` 是加载点沿正式方向的准静态位移增量；
- `E_C3` 是不丢失 B 单元内部事件组的跨单元事件/级联记录；
- `M_C3` 包含首个针失效、首个单元显著退化、最弱分支、稳定反力峰值、峰后分支和终止证据；
- `Sigma_C3` 必须区分平衡接受、事件缩步、退化 graph、合同/运动未认证、物理无平衡、物理失稳、不可恢复整体脱附、数值失败和事务失败。

本轮必须形成：

1. C2 完整历史到 C3 的无损继承和持续加载规则；
2. 针级失效/滑移/脱离/损伤事件到单元能力退化的传播接口；
3. 可使用低维单元响应的严格条件和强制完整 B 回调条件；
4. 任一单元事件后的四单元非线性再平衡、姿态更新和载荷重分配；
5. 一侧压紧、对侧剥离及 `+X/45°` 两方向的分支作用差异；
6. 首个针失效、首个单元显著退化、整体稳定峰值、物理失稳和不可恢复脱附的互不混淆定义；
7. `F_reaction(delta_P)` 稳定分支、整体最大承载 `F_crit` 和峰后终止的可执行定义；
8. 事件定位、跨单元同时事件、共享 DamageStore fixed point、同位置级联、稳定性复核和全局原子提交的完整循环；
9. 当前 B 合同扩展缺口在 C3 中的精确安全边界；
10. 面向未来偏心拉力实验的完整输出合同、验证矩阵和大模块 C 集成交接。

## 4. 必须解决的 C3 问题

### 4.1 偏心载荷下的一侧压紧、对侧剥离

必须从 C2 已接受的符号和完整 wrench 出发解释，而不是凭示意图判断：

- `+X` 正向加载在 C 点产生 `+M_Y=50F N·mm`；contact/authorized wrench 需要提供相反力矩；在忽略其他项的符号检查中，这会使 X 对置分支出现法向差和一侧靠墙、一侧离墙；
- `45°` 载荷同时产生 `M_X=-50F/sqrt(2)` 与 `M_Y=+50F/sqrt(2)`，同时耦合 X/Y 两组对置分支；
- rocking、`rho_A/i`、单元自身自由力偶、横向力、作用线、历史损伤和执行器边界会共同影响实际压紧/剥离；
- `P_i` 恒定不等于各单元实际法向接触合力恒定；靠墙一侧可能增载，离墙一侧可能释放、滑移、失效或再挂接；
- 每次姿态或事件变化后，具体分支必须由完整 B response 和整体六维平衡确认，不得把法向位移趋势直接升级为单元失败结论。

必须给出可检查的逐步链：偏心外载增加 → 整体姿态/法向位置调整 → B 内针级载荷和活动集变化 → 单元 wrench/graph 改变 → 四单元重新平衡 → 下一稳定状态或事件。

### 4.2 单元级状态压缩接口

C3 必须定义一个历史相关、可回调的单元状态压缩接口。它至少包含：

- 原始 `EmbeddedUnitTrialResponse` 或不可丢失的完整句柄；
- 当前 contact-only wrench、参考点、表达框架、自由力偶、中心轴/作用点可用性和功检查；
- 当前 B/A accepted state 与 opaque 历史、DamageStore 版本和依赖；
- balance/graph、branch、rank、nullspace、条件指标和认证等级；
- 当前活动/承载针摘要、粘滑/材料/针体/硬限位/再挂接状态及原始事件句柄；
- 局部一侧切线、割线或 admissible graph、trust region、适用方向和事件括区间；
- 逐针/单元剩余弹簧行程、合法路径、强度/损伤/碰撞/域裕度和恢复能力；
- 当前最弱归一化裕度、残余方向能力和不确定性；
- `full_unit_resolve_callback_requirement` 与理由码。

该接口不能退化为单元峰值、固定极限面、单个切线刚度、有效刺数或平均单刺力。低维摘要只用于事件外局部预测、步长控制和最弱分支审计；所有历史和完整回调条件必须保留。

### 4.3 针级失效到单元能力显著退化的传递

必须区分至少以下事件：

1. 首个针的摩擦滑移、材料失效、针体强度事件、硬限位、接触释放或再挂接；
2. 单元内部同时事件和同位置级联；
3. 单元 contact-only wrench、局部 graph、切线、作用线、活动承载集合或剩余行程的显著变化；
4. 单元当前无承载但可恢复，与已经不可恢复脱附；
5. 单元能力显著退化事件。

“首个针失效”不得自动等于“单元失败”。每个针级事件必须通过 B 完整回调、单元内部事件后重平衡和共享 DamageStore 协调传播到单元响应。只有当前单元的版本化能力/稳定性通道跨越明确定义的事件面时，才能记录 `FIRST_UNIT_SIGNIFICANT_DEGRADATION` 或等价事件。

显著退化判据必须是可配置、可标定和有单位/归一化尺度的事件函数族，例如当前最弱裕度越过零、残余方向能力下界越过门槛、承载 graph 失去当前方向、单元由可恢复转不可恢复、或关键 wrench/刚度通道在事件前后出现经验证的有限跳变。不得硬编码唯一百分比或用论文数值；阈值缺失时事件应为 `UNAVAILABLE/UNCERTIFIED`，同时仍保留原始连续量和针级事件。

### 4.4 单元失效后的四单元重新平衡

任一单元/针级事件后必须：

1. 定位共同加载路径上的最早事件点并拒绝跨事件目标；
2. 从同一全局 accepted state、同一 C2/C3 位姿和同一 DamageStore 基线重调四单元；
3. 协调跨单元共享损伤并形成一致 trial DamageStore；
4. 重新求整体允许位姿、`lambda_P`、可选执行器相对行程、四单元 admissible graph 和六维平衡；
5. 重新检查无隐藏支承、功、残量、稳定性、行程、碰撞、域、模型和参数；
6. 若新平衡触发同位置的其他针/单元事件，继续级联直到 fixed point 或明确终止；
7. 只有一致、认证且稳定的候选才能全局原子提交。

载荷重分配必须由位移兼容、状态相关刚度/graph、接触约束、整体姿态和六维平衡自动产生。禁止使用：

- 失效单元旧载荷均分给其余三个单元；
- 最近邻/距离权重或固定分配矩阵；
- 文献 23 的 `F_7/6` 作为正式规则；
- 四个单元峰值或旧切线按比例缩放；
- 只重算触发事件的单元而冻结其余三个单元。

文献 23 的等刚度换载只能作为特殊解析基准：在构造的等刚度、等增量、固定姿态、无其他约束变化极限中，本项目重求解应退化出相同方向的等增量结果；离开该极限即必须使用完整模型。

### 4.5 低维单元响应与完整针级重求解的切换

只有以下条件全部满足时，C3 才可在返回 trust region 内使用单元局部切线、割线或 graph：

- 当前 B 扩展已认证请求的完整基座 twist/姿态；
- 参考点、表达框架、控制模式、`P_i`、系统边界和 DamageStore 与返回量一致；
- 活动支持、粘滑、弹簧、材料、强度和恢复分支不变；
- accepted states、表面、参数、姿态和变换版本不变；
- 预测增量不跨越任何针级、单元级、整体稳定性、姿态 trust region、碰撞或域事件；
- graph/切线质量合格且所需保守下界存在；
- 不需要精确逐针损伤冲突、级联、作用点、能量或恢复判断。

满足任一以下条件必须完整回调 B：

- 预测或实际接触建立/释放、滑移、支持迁移、材料/针体失效、软化、硬限位、脱离或再挂接；
- 活动集、分支、DamageStore、accepted state、姿态、控制模式或系统边界改变；
- graph 非唯一、切线奇异/不可用、超出 trust region；
- 接近碰撞、域边界、几何质量或小角度认证边界；
- 单元显著退化候选、整体峰值附近、峰后分支、稳定性/恢复性判断；
- `full_unit_resolve_callback_requirement=true`。

### 4.6 事件分类与互不混淆的里程碑

必须定义下列事件记录的触发函数、事件前/点/后状态、是否提交、证据和重复发生语义：

- `FIRST_NEEDLE_FAILURE`：本次 C3 路径中第一个经 B/A 认证的针级材料/强度或等价失效事件；滑移、硬限位和接触释放应分别记录，不能全部叫材料失效；
- `FIRST_UNIT_SIGNIFICANT_DEGRADATION`：第一个通过版本化单元事件函数的显著退化；
- `GLOBAL_REACTION_PEAK_CANDIDATE`：当前稳定已接受分支上的局部/区间峰值候选；
- `GLOBAL_CRITICAL_CAPACITY_CONFIRMED`：全过程终止或已证明无更高合法稳定分支后确认的 `F_crit`；
- `GLOBAL_EQUILIBRIUM_INFEASIBLE`：合同、模型、参数和数值有效，但当前加载位置无 admissible 六维代数平衡；
- `GLOBAL_PHYSICAL_INSTABILITY`：代数根可存在，但所有 admissible 一侧稳定分支消失；
- `GLOBAL_DETACHMENT_RECOVERABLE`：当前整体或关键分支失去承载，但存在合法继续位移/再挂接稳定路径；
- `GLOBAL_DETACHMENT_IRRECOVERABLE`：已证明在允许路径、行程和状态空间内没有可继续稳定承载/再挂接分支；
- 合同/运动/模型/参数/域/碰撞/几何/数值/事务失败的独立状态。

首个针失效可能早于整体峰值；首个单元显著退化也可能仍可由其余分支重新平衡；反力局部下降不自动等于物理失稳；Newton 失败不自动等于无平衡；未认证 B 运动不自动等于脱附。

### 4.7 最大承载和稳定反力曲线

每个 accepted 稳定步输出：

$$
F_{\mathrm{reaction}}(\delta_P)=\lambda_P
$$

及其加载方向、作用—反作用约定、整体位姿、四单元 wrench 和稳定分支 ID。最大承载定义为当前正式方向、固定初态和配置下所有**经认证可达稳定分支**上的反力上确界/最大值：

$$
\boxed{
F_{\mathrm{crit}}
=
\max_{\delta_P\in\mathcal B_{\mathrm{stable,reachable}}}
F_{\mathrm{reaction}}(\delta_P)
}
$$

其中 `B_stable,reachable` 必须由连续 accepted 历史、事件跳转和允许的分支选择规则定义；不能把未认证点、数值失败点、隐藏支承平衡、跨事件插值点或单元峰值和加入集合。

必须处理：

- 非光滑峰值、平台、事件点左右极限和多个局部峰；
- 位移控制下峰后仍可能存在的稳定下降分支；
- graph 非唯一时的保守集合报告、分支标识和无法唯一确认峰值的状态；
- 峰值事件定位、步长收敛、反力/位移容差和不确定性；
- `peak_candidate` 与最终 `F_crit_confirmed` 分开；没有完成合法终止/搜索证据时不得把当前最大观测值冒充最终能力；
- `+X` 与 `45°` 必须分别从可比较的初态和样本生成各自曲线，不能混用路径历史。

### 4.8 峰后失稳和终止条件

首个峰值或反力下降本身不允许终止。位移控制应在安全、模型有效和存在稳定可达分支时继续，以捕捉渐进剥离、再平衡、再挂接、二次峰和最终脱附。

物理终止至少区分：

1. 经事件定位后当前加载路径不存在任何 admissible 六维平衡；
2. 所有可达平衡分支在允许的一侧扰动下失去稳定性；
3. 关键单元/整体进入不可恢复脱附，且剩余合法行程、活动针、再挂接和其他分支无法形成稳定平衡；
4. 达到工程几何/结构允许边界前已定位的物理终点。

以下情况不能作为物理峰后终止：

- `C3_CONTRACT_EXTENSION_REQUIRED/KINEMATIC_MODE_UNSUPPORTED`；
- `MODEL_UNAVAILABLE/PARAMETER_UNAVAILABLE`；
- `OUT_OF_DOMAIN/GEOMETRY_UNCERTAIN/BODY_COLLISION_INVALID`；
- `STABILITY_UNCERTIFIED`；
- `NUMERICAL_NONCONVERGENCE`；
- 事务失败、陈旧快照或哈希不一致。

这些状态应保留最后有效 accepted state、当前峰值候选和“最终能力未确认”标志。工程事实允许失去准静态稳定后终止，不要求模拟脱落后的大角度运动。

### 4.9 `+X` 与 `45°` 的四分支作用差异

必须在理想四重对称和实际历史相关两层分析：

- 理想相同单元、各向同性表面、相同 `P_i`、相同预紧/损伤/行程下，轴向方向可由 90° 旋转对应；
- `+X` 主要对 X 对置组施加倾覆力矩，但 Y 组仍通过横向能力、整体平衡、姿态和作用线参与；不能先验设其载荷不变；
- `45°` 同时耦合 X/Y 两组，可能分散载荷，也可能让两组同时接近剥离/横向边界；
- 方向性粗糙表面、`2×5/5×2` 阵列、不同事件、DamageStore、作用点、graph、剩余行程和 `rho_A/i` 会破坏对称；
- 必须保存四单元原始 wrench、姿态、最弱裕度、首失效身份、事件顺序和峰后路径；不得只比较两个总峰值；
- 最终哪个方向更有利只能由同一设计、成对表面/随机种子、同一 C1 策略和等价初态的完整求解得出。本轮不得未经求解固定排序。

### 4.10 整体能力域的最小实现

C3 最小能力实现不是预先构造一个永久凸多面体，而是：

1. 在当前 accepted 历史和活动集上，由四个 B 历史相关 response/graph 装配局部 admissible 整体 wrench 集；
2. 在正式加载方向和位移控制约束下，求当前最弱归一化裕度/残余能力；
3. 把事件距离、trust region、姿态和 DamageStore 依赖绑定到该局部能力对象；
4. 一旦事件、历史、姿态或活动集变化，立即失效并完整重求；
5. 始终保留原始四单元 response，不能只保留一个裕度 `c`。

可借鉴文献 28 的“最大化最小裕度”结构，但能力边界必须来自 B，且应允许非线性、非凸、集合值和历史依赖。若只能形成局部保守内近似，必须给出有效域、遗漏风险和完整回调条件，不能声称全局能力域已知。

### 4.11 面向未来偏心拉力实验的输出合同

每个方向和样本至少输出：

- 设计、表面类别/实现/随机种子、参数包、工程事实/B 合同/C 上下文版本；
- `C1PreloadState_id`、`s_stop`、四个 `P_i`、预紧事件和初始 DamageStore；
- 加载方向、加载点位置、位移控制轨迹、物理位移/时间映射是否可用；
- `delta_P`、`F_reaction`、完整六维反力/外部 wrench、整体位姿和 rocking 角；
- 四单元 contact-only force/moment、作用线/压力中心可用性、实际法向/切向分量、最弱裕度和剩余行程；
- 针级和单元级事件、事件位移/反力、同时事件组、损伤、级联、再挂接和恢复性；
- 首个针失效、首个单元显著退化、峰值候选、确认的 `F_crit`、峰后终止类型和最后稳定状态；
- 功/能量/耗散/数值误差、残量、稳定性、质量、认证、版本、哈希和提交收据；
- 原始连续曲线和事件记录，不输出未经定义的二元“成功/失败”或综合评分。

必须给出仿真—实验对齐规则：加载点和方向、位移零点、预紧初态、传感器作用—反作用、滤波/采样、事件时间对齐和重复样本统计。当前验证目标首先是趋势、设计排序和机理解释，不要求硬拟合单次偶然峰值。

### 4.12 C3 数据模型与单步状态机

必须定义或等价完整描述：

- `C3LoadRequest`：当前 `C2AcceptedState`、加载方向和增量、模式/约束、四个 `P_i`、B 合同/扩展版本、四单元 accepted snapshots、变换、DamageStore、事件/稳定性/峰值/数值配置和幂等性键；
- `C3LoadTrialResponse`：四个原始 B responses、wrench、整体残量/graph、反力/位姿、事件、损伤、重平衡、稳定性、峰值候选、失败码和事务意图；
- `C3AcceptedState`：累计路径、当前稳定分支、四单元/B/A/DamageStore 状态、事件历史、当前最大观测稳定反力、峰值候选、是否已确认、功/哈希/收据和下一回调条件；
- `C3EventRecord`：事件层级、实体、类型、括区间、前/点/后状态、同时组、因果依赖、DamageStore 版本、是否提交和恢复性；
- `C3MaximumCapacityResult`：方向、初态、`F_crit` 或不可确认状态、峰值位置/分支、终止证据、四单元临界载荷分配、首失效里程碑、误差/不确定性和完整曲线句柄。

单步算法必须从无副作用试探到全局原子提交闭环，不能只给静力方程或 LP。

## 5. C3 单步可执行流程

至少用有序步骤或伪代码覆盖：

1. **验证并冻结**：校验当前 accepted state、`s_stop`、`delta_P`、四个 B/A snapshots、DamageStore、变换、方向、模式、合同版本、配置和幂等性键；缺失字段不得默认补齐。
2. **提出共同加载目标**：生成新的加载点位移目标和允许整体位姿/执行器行程候选；不改变内部 `s_stop`。
3. **运动合同覆盖审计**：计算四单元完整基座 twist；未认证局部 y/姿态时安全拒绝，不推进历史或峰值确认。
4. **四单元无副作用完整/局部试探**：按第 4.5 节决定局部预测或完整 B 回调，收集 response/graph、wrench、切线、事件、损伤意图、作用点和 rollback token。
5. **wrench 运输和功检查**：在当前动态姿态下把四个 contact-only wrench 运输到 C，检查参考点和功不变。
