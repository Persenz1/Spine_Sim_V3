## 1. 本轮唯一目标

请把已经正式接受的 A、B、C 三个大模块集成为一份完整、一致、可实现、可验证的系统机理规范 `SYSTEM_INTEGRATED_MODEL.md`。本轮不是把三份文件顺序拼接，也不是重新推导单刺、阵列或整爪低层方程，而是建立一个从表面实现到单刺、阵列单元、十字对爪和实验输出的唯一全局依赖链，使以下内容在系统层闭合：

- A→B→C 的变量、状态、历史、事件和事务传递；
- 全局/单元/针/接触坐标、参考点和刚体变换；
- mm、N、N·mm、s、rad、MPa、N/mm 等单位；
- 接触力、`A_on_B`、单元 contact-only wrench、加载器作用—反作用和功方向；
- 恒主动法向推力、径向驱动、位移控制反力、约束反力和墙面接触 wrench 的唯一所有权；
- 预载、搜索、接触、滑移、损伤、脱离、再挂接、重分配、渐进失效、峰值和终止的事件关系；
- trial、rollback、prepare、commit、共享 `DamageStore` 和 accepted history 的原子语义；
- 单刺/单元连续拖拽与整爪同步预紧—偏心承载之间的路径和终止边界；
- 实验验证所需的原始连续输出、事件历史、质量和不确定性；
- 当前已经闭合的规范、尚未闭合的实现/合同/参数问题及其精确交接。

最终文件必须能直接指导后续求解器数据模型、调用顺序、残量装配、事件循环、事务实现和验证测试，同时不得复制或越权重定义低层机理。

## 2. 研究对象、物理背景与系统位置

本项目研究钩爪式爬壁机器人的爪刺啮合。目标表面为红砖、混凝土和不同目数砂纸；主几何表示为 `150 mm × 150 mm` 单值高度场，并保留完整三角网格能力分支。只有局部球形针尖参与承载，锥段、针杆和安装座只进行禁止碰撞与可装配检查。

从低到高的物理依赖固定为：

\[
\boxed{
\text{SurfaceRealization/A1 查询}
\rightarrow
\text{A 单刺本征算子}
\rightarrow
\text{B 阵列单元共同平衡算子}
\rightarrow
\text{C 十字对爪同步预紧与偏心承载算子}
\rightarrow
\text{原始仿真—实验输出}
}
\]

- A 负责一根针沿连续路径的有限球尖几何、接触、摩擦、结构柔顺、材料容量、损伤、针体强度、释放和再挂接。
- B 负责规则多刺阵列在共同刚性背板运动下的部分接触、恒单元主动推力平衡、活动集、真实载荷共享、事件后重分配和单元历史响应。
- C 负责四个单元的共同径向搜索、预紧停止与锁定、整爪刚体运动、偏心加载、四单元 wrench 装配、渐进事件、稳定分支、峰值和全局原子事务。
- 系统层只协调依赖、接口、全局状态、事件、事务、验证和实现交接；不得在系统层新写一套球尖接触、摩擦、材料强度、针梁、针级弹簧、阵列载荷共享或低层损伤演化。

工程目标首先是比较参数变化方向、设计方案排序和机理解释，而不是拟合一次偶然峰值或自动给出二元抓附成功。所有层级必须保留原始连续量和事件，供未来实验阈值与综合指标另行定义。

## 4. 必须保留的 accepted 边界

### 4.1 A→B 边界

- B 的唯一 A 物理入口是 `embedded_constitutive_trial` / `intrinsic_single_spine_kernel` 语义；A 的 standalone driver 只用于 A 层独立验证。
- A 的公共 wrench 方向是单刺 A 子系统对背板 B 的 `A_on_B`，且带表达坐标和参考点。
- A embedded 请求中不得出现逐针 `0.5 N` 法向载荷。单刺 `0.5 N` 只属于 standalone A driver；B 每单元 `0.5–2 N` 主动推力由 B 外层施加一次。
- A 试探无永久副作用；B 只调度共享损伤冲突，不能改写 A 的材料律或 opaque 状态。

### 4.2 B→C 边界

- C 的唯一 B 物理入口是无副作用的 `embedded_array_unit_trial`；C 不得直接调用 A，也不得绕过 B 的全阵列共同平衡。
- B 的公共 `contact_only_wrench` 是全部针 `A_on_B` 净 wrench 运输到 `O_A` 后的和；针梁、针级弹簧、硬限位和接触力不得再加第二次。
- (P_i)、x 位移控制反力和 y/转动约束反力必须与 contact-only wrench 分栏。当前整爪墙面外部装配策略唯一为 `CONTACT_ONLY_EXTERNAL_ASSEMBLY_V1`。
- `UnitCapabilityState` 是绑定历史、姿态、分支、`DamageStore` 和 trust region 的局部算子状态，不是无记忆极限面。
- 事件、姿态、活动集、控制模式、合同、参考点或 `DamageStore` 变化后必须按合同完整回调，不得旋转、缩放或外推旧 wrench。

### 4.3 当前全局运行能力缺口

必须原样保留并在系统闭合结论中突出以下事实：

- `B_TO_C 1.0.0 accepted` 只认证单元局部 x 与全局 Z 平移；不认证局部 y、真实单元转动或完整 SE(3) twist。
- C 的同步预紧 x/Z 路径在当前合同内可表达；C 的偏心加载理论、六维平衡、稳定性、事件和峰值规范已经集成。
- 但正式非零 `+X` 路径会使部分单元需要局部 y，`45°` 路径会使全部单元需要局部 y；rocking 还需要动态姿态和旋转查询。因此在 B 1.0 下正式非零偏心加载必须返回 `C_CONTRACT_EXTENSION_REQUIRED`，accepted 状态、δP、B/A 历史、`DamageStore`、事件、功、曲线和峰值均零推进。
- 该安全拒绝不等于零承载、物理无平衡、物理失稳、不可恢复脱附、数值失败或抓附失败。
- 系统模型可以整理并冻结“实现交接所需的 B 2.x 最小扩展要求和验证门槛”，但不得把尚未接受的 B 2.x 写成当前正式合同，也不得通过 x/Z 投影、旋转旧 wrench、固定针姿态或经验能力域伪装闭合。

因此必须分别报告：

1. 理论/规范依赖链是否全局一致；
2. 当前 accepted 合同下哪些阶段可在线执行；
3. 哪个接口缺口阻断正式偏心运行；
4. 关闭缺口所需的最小版本化扩展、验证和接受条件。

不得用一句“系统已完全闭合”掩盖上述等级差异。

## 8. 统一求解与验收要求

### 8.1 全局装配

- 每个 A 净 wrench 在 B 中只运输/求和一次；每个 B contact-only wrench 在 C 中只运输/求和一次。
- 所有变换必须携带源/目标坐标、源/目标参考点、版本和单位；只旋转力而遗漏力矩运输为错误。
- (P_i) 是恒主动广义推力，不是恒实际接触合力，也不是逐针载荷；不得在 C 外部墙面接触总和中再加一次。
- 固定姿态不自动授权外部支承反力矩；未授权乘子必须为零或候选拒绝。

### 8.2 状态与历史

- 系统 accepted state 必须把 C、四个 B、全部 A opaque states、一个共享 DamageStore、事件/能量/曲线账本和提交收据绑定为一个版本组合。
- 任何层级的 trial 都不能永久修改 accepted history；跨层提交要么全成功，要么全回滚。
- 新独立试验/新表面 realization 从无损开始；同一连续路径中的损伤、路径、事件和耗散不得在释放或再挂接后重置。

### 8.3 事件和终止

- 所有候选事件都用原始有量纲量、括区间、事件分数和一侧状态保存；数值归一化不能取代物理阈值。
- 物理事件按合法路径最早位置处理；依赖偏序只规定后侧重算顺序，不决定并发事件的先后。
- 首针失效不等于单元失败；首单元退化不等于整体峰值；峰值不等于物理终止；当前失载不等于不可恢复脱附。
- 只有合同、模型、参数、域、数值、稳定性和分支覆盖均充分时，才能把无解、失稳、不可恢复或 (F_{\rm crit}) 当作物理结论。

### 8.4 证据和未决数值

本轮输入已经包含正式机理及其证据边界。不得为追求“完整”而搜索或硬编码新的材料数值。通用坐标、虚功、残量、事务或验证表达可以作为集成数学使用，但必须标为系统集成推导，不能升级为工程事实或材料证据。任何缺失数值都保留为参数、配置 ID、范围、候选模型或 `unavailable`。
